from flask import Flask, request, jsonify
import joblib
import numpy as np

app = Flask(__name__)

# Load model
model = joblib.load("models/placement_model.pkl")
scaler = joblib.load("models/scaler.pkl")

@app.route('/predict', methods=['POST'])
def predict():
    data = request.json  # Get input data
    features = np.array([data['features']])
    features = scaler.transform(features)  # Scale input data
    prediction = model.predict(features)
    return jsonify({"placement": "Placed" if prediction[0] == 1 else "Not Placed"})

if __name__ == '__main__':

Machine Learning Algorithms Used in the Placement Prediction Application

This project primarily focuses on binary classification (whether a student is placed or not). Various machine learning algorithms are used to improve model performance. Below are the algorithms used and their respective use cases in the project.

1. Logistic Regression

Use Case in the Project:

  • Used for baseline classification (placed or not placed).
  • Works well for linearly separable data, where placement depends on factors like CGPA, programming skills, and certifications.
  • Provides interpretability by showing the influence of each feature on placement.
python
from sklearn.linear_model import LogisticRegression

model = LogisticRegression()
model.fit(X_train, y_train)

✅ Best for: Basic classification tasks, easy to interpret.
❌ Limitation: Does not handle complex patterns well.

2. Decision Tree Classifier

Use Case in the Project:

  • Used to capture non-linear relationships in data.
  • Helps identify key decision-making features (e.g., “If CGPA > 8.0 and Certifications > 2, then Placed”).
  • Easily explainable as a tree structure.
python
from sklearn.tree import DecisionTreeClassifier

model = DecisionTreeClassifier(max_depth=5)
model.fit(X_train, y_train)

✅ Best for: Interpretable decision rules, handling missing values.
❌ Limitation: Prone to overfitting on training data.

3. Random Forest Classifier

Use Case in the Project:

  • An ensemble model that combines multiple decision trees for better accuracy.
  • Reduces overfitting compared to a single decision tree.
  • Helps identify the importance of each feature in placement prediction.
python
from sklearn.ensemble import RandomForestClassifier

model = RandomForestClassifier(n_estimators=100, random_state=42)
model.fit(X_train, y_train)

✅ Best for: Handling complex data patterns, reducing overfitting.
❌ Limitation: Computationally expensive.

4. Support Vector Machine (SVM)

Use Case in the Project:

  • Used when the dataset has high-dimensional features (e.g., CGPA, skills, internships).
  • Uses a hyperplane to classify students into "Placed" and "Not Placed."
  • Works well in cases where data is not perfectly separable.
python
from sklearn.svm import SVC

model = SVC(kernel='rbf', probability=True)
model.fit(X_train, y_train)

✅ Best for: Handling complex, high-dimensional data.
❌ Limitation: Slow for large datasets.

5. XGBoost (Extreme Gradient Boosting)

Use Case in the Project:

  • Best-performing model for placement prediction.
  • Works well on imbalanced datasets where more students are "Not Placed" than "Placed."
  • Uses boosting techniques to improve weak predictions.
python
import xgboost as xgb

model = xgb.XGBClassifier(n_estimators=100, learning_rate=0.1)
model.fit(X_train, y_train)

✅ Best for: High accuracy, handling imbalanced data, fast training.
❌ Limitation: Requires fine-tuning for optimal performance.
